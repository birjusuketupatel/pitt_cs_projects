test case: delete-abort
description:
  verifies that a delete of an existing table from an aborted transactions does not affect the state of the database
instructions:
  delete existing tables with "rm -rf tables"
  run "./pittRCDB roundrobin ./tests/recovery/normal/delete-abort/delete-abort.txt"
  save result of the final operation "T Starbucks"
  delete created tables with "rm -rf tables"
  run "./pittRCDB roundrobin ./tests/recovery/normal/delete-abort/delete-no-abort.txt"
  save result of the final operation "T Starbucks"
expected behavior:
  after the first operation, the table should contain 4 records
  after the second operation, the table should be empty
